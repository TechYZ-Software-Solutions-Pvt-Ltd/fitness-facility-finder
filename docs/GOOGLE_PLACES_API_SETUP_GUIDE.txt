GOOGLE PLACES API SETUP GUIDE
=====================================

Complete Step-by-Step Guide for Google Places API Key Setup
For Individual Google Accounts and Company Google Accounts

=====================================
QUICK START - FOR NON-TECHNICAL USERS
=====================================

-------------
This guide helps you get a "key" that allows the Facility Search app to find
gyms, fitness centers, and other facilities near you using Google's database.

YES! Google gives you $200 worth of free searches every month.
That's enough for 11,000+ facility searches - perfect for personal use!

- Individual accounts: 10-15 minutes

- No technical knowledge required!

WHAT YOU NEED:

✓ A Google account (Gmail, Google Workspace, etc.)

✓ 15 minutes of your time

1. Go to Google Cloud Console

=====================================
FOR COMPANY/ORGANIZATION ACCOUNTS
=====================================

STEP 1: ACCESS GOOGLE CLOUD CONSOLE
------------------------------------
1. Go to https://console.cloud.google.com/
2. Sign in with your company Google account
3. Select your organization from the account dropdown (top-right)
4. If you don't have a project, click "Select a project" → "New Project"

STEP 2: CREATE OR SELECT A PROJECT
----------------------------------
1. Click "New Project" (if creating new)
2. Enter project name: "Facility Search App" (or your preferred name)
3. Select organization (if applicable)
4. Click "Create"
5. Wait for project creation (usually 1-2 minutes)

STEP 3: ENABLE BILLING
----------------------
1. Navigate to "Billing" in the left sidebar
2. Click "Link a billing account"
3. Create a new billing account or link existing one
4. Add payment method (credit card, bank account, etc.)
5. Verify billing is active

STEP 4: ENABLE PLACES API
-------------------------
1. Go to "APIs & Services" → "Library"
2. Search for "Places API"
3. Click on "Places API"
4. Click "Enable"
5. Wait for activation (usually instant)

STEP 5: CREATE API CREDENTIALS
------------------------------
1. Go to "APIs & Services" → "Credentials"
2. Click "Create Credentials" → "API Key"
3. Copy the generated API key (starts with "AIza...")
4. Click "Restrict Key" (recommended for security)

STEP 6: CONFIGURE API KEY RESTRICTIONS (RECOMMENDED)
----------------------------------------------------
1. Click on your API key to edit it
2. Application restrictions:
   - Select "HTTP referrers (web sites)"
   - Add your domain: localhost:8501, yourdomain.com
3. API restrictions:
   - Select "Restrict key"
   - Choose "Places API"
4. Click "Save"

=====================================
FOR INDIVIDUAL GOOGLE ACCOUNTS
=====================================

STEP 1: ACCESS GOOGLE CLOUD CONSOLE
------------------------------------
1. Go to https://console.cloud.google.com/
2. Sign in with your personal Google account
3. Accept terms if prompted
4. Click "Select a project" → "New Project"

STEP 2: CREATE A PROJECT
------------------------
1. Enter project name: "My Facility Search App"
2. Leave organization as "No organization"
3. Click "Create"
4. Wait for project creation

STEP 3: SET UP BILLING
----------------------
1. Go to "Billing" in the left sidebar
2. Click "Link a billing account"
3. Create a new billing account
4. Add your payment method
5. Verify billing is active

STEP 4: ENABLE PLACES API
-------------------------
1. Go to "APIs & Services" → "Library"
2. Search for "Places API"
3. Click on "Places API"
4. Click "Enable"
5. Wait for activation

STEP 5: CREATE API KEY
----------------------
1. Go to "APIs & Services" → "Credentials"
2. Click "Create Credentials" → "API Key"
3. Copy the generated API key
4. Click "Restrict Key" (highly recommended)

STEP 6: SECURE YOUR API KEY
---------------------------
1. Click on your API key to edit it
2. Application restrictions:
   - Select "HTTP referrers (web sites)"
   - Add: localhost:8501, 127.0.0.1:8501
3. API restrictions:
   - Select "Restrict key"
   - Choose "Places API"
4. Click "Save"

=====================================
SECURITY BEST PRACTICES
=====================================

1. API KEY RESTRICTIONS
-----------------------
- Always restrict your API key
- Use HTTP referrers for web applications
- Limit to specific APIs (Places API only)

2. MONITOR USAGE
----------------
- Set up billing alerts in Google Cloud Console
- Monitor API usage regularly
- Set spending limits if needed

3. KEY ROTATION
---------------
- Rotate API keys periodically
- Use different keys for development and production
- Never commit keys to version control

-------------------------

✓ Upgrade billing plan if needed

PROBLEM: "Request Denied"

✓ Check API restrictions

✓ Ensure Places API is enabled

----------------------------

✓ Make sure business type is selected
✓ Verify your API key is working
